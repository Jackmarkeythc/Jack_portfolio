Physical Interface Project 3 — Meta Quest Unity Game Development

Role: Ideation lead, poster designer, gameplay flow scripting (scene/“frame” transitions) & background‑music design.

Platform: Meta Quest (standalone) · Engine: Unity (2022 LTS) · Tooling: XR Interaction Toolkit / Oculus Integration, Unity AudioMixer, Timeline, Addressables.

Team: Course project (cross‑functional: design × dev). Timeframe: Sprint prototype → iterative polish.

1) Executive Summary

We designed and built an interaction‑first VR prototype for Meta Quest that guides players through a sequence of “frames” (scenes) with seamless audiovisual continuity. I led concept direction, poster design, and the core scripting that connects frames, manages state, and crossfades background music so transitions feel intentional rather than jarring. The result: a playable Quest build that onboards first‑time users quickly, maintains comfort, and communicates the concept clearly via a production‑ready poster.

2) Problem & Objectives

Challenge: Early VR prototypes often feel like disconnected scenes stitched together, with abrupt audio cuts and unclear navigation. We needed a cohesive through‑line that preserves immersion on a standalone headset.

Objectives:

  • Cohesion: Connect frames with a clear state machine so the user always knows “where they are”.
  • Continuity: Background‑music system that crossfades themes between frames (no dead air, no snaps).
  • Comfort: Teleport/short‑hop movement, reduced vection, predictable interactions.
  • Clarity: One‑page poster for gallery/crit prompt communication.

Constraints: Standalone performance budget (72–90 Hz target), limited sprint time, minimal external assets to keep build size lean.

3) Concept

We framed the experience as a guided path of frames—each frame introduces one new interaction idea, then resolves into the next with a musical hand‑off and a visual cue (fade/portal). The flow is a sequence of vignettes where users progress with clear, lightweight affordances.

  1. Frame A – Onboarding: safety bubble, controller prompts, single interaction.
  2. Frame B – Interaction Depth: 2–3 affordances that chain (grab → place → confirm).
  3. Frame C – Payoff: brief culmination moment (reveal/vista/effect) that transitions to credits/loop.

4) Process

Principles: comfort‑first locomotion, affordance mapping, and a feedback triad (haptic + spatial audio + visual highlight).

Flow: [Start] → Onboard → Interact → Payoff → End/Loop. State is explicit (Onboarding, Interact, Payoff, End) and each transition triggers both scene content and audio theme changes.

5) Technical Architecture & Implementation

High level: Single master scene with additive sub‑scenes for frames (fast loads, persistent managers). Lightweight state machine governs flow; Addressables handle scene assets; AudioMixer and snapshots manage music crossfades.

5.1 Frame‑to‑Frame Scripting

I authored a compact flow controller that emits events for UI, level content, and music to stay in lockstep (see code excerpts).

// FrameId enum + GameFlowController (conceptual)
public enum FrameId { Onboard, Interact, Payoff, End }

public class GameFlowController : MonoBehaviour {
  public FrameId Current { get; private set; } = FrameId.Onboard;
  public event Action OnFrameChanging;
  public event Action OnFrameChanged;

  public void Advance() {
    var next = NextOf(Current);
    if (next == Current) return;
    OnFrameChanging?.Invoke(Current, next);
    Current = next;
    OnFrameChanged?.Invoke(Current);
  }

  FrameId NextOf(FrameId f) => f switch {
    FrameId.Onboard => FrameId.Interact,
    FrameId.Interact => FrameId.Payoff,
    FrameId.Payoff => FrameId.End,
    _ => FrameId.End
  };
}
5.2 Music Crossfades

The music system uses two alternating AudioSources so theme handoffs are always crossfaded rather than hard‑cut. This preserves a single‑take feel even as content swaps in the background.

// MusicDirector (conceptual)
public class MusicDirector : MonoBehaviour {
  [SerializeField] AudioSource a; // current
  [SerializeField] AudioSource b; // next
  [SerializeField] float fadeSeconds = 2.5f;

  public void PlayTheme(AudioClip clip) {
    var next = !a.isPlaying || a.volume < b.volume ? a : b;
    var prev = next == a ? b : a;
    next.clip = clip; next.volume = 0f; next.loop = true; next.Play();
    StartCoroutine(Crossfade(prev, next));
  }

  IEnumerator Crossfade(AudioSource prev, AudioSource next) {
    float t = 0f;
    while (t < fadeSeconds) {
      t += Time.deltaTime;
      float k = t / fadeSeconds;
      next.volume = k; prev.volume = 1f - k;
      yield return null;
    }
    prev.Stop();
  }
}

6) Interaction & Comfort Defaults

  • Teleport by default; snap‑turn 30°.
  • Dynamic handedness based on system settings; mirrored prompts.
  • Soft vignette during fast moves; player height calibration.

7) Sound & Music Design

Ambient pads + light percussive ticks; BPM ~100–120. Themes are loop‑safe and matched to bar lengths; transitions occur on bar boundaries with optional low‑pass sweeps for polish.

8) Poster & Communication Artifact

Poster summarizes concept, controls, and flow for gallery critiques: 3‑column grid, headline, frame strip diagram, QR to build/video, controller map, and comfort badges. Deliverables: print PDF + screen PNG.

9) Testing & Iteration

Classroom playtests validated frame clarity and audio continuity. The main early issue—users missing the Advance affordance—was fixed by adding a persistent continue cue and a brief haptic nudge. Profiling (Frame Debugger/Profiler) informed batching and culling optimizations.

10) Outcomes

  • Playable Quest build with stable transitions and smooth music handovers.
  • Clear communication via poster and in‑app hints; faster onboarding.
  • Reusable architecture (flow controller + music director) for future projects.

11) Next Steps

  • Add hand‑tracking fallback for controller‑less demos.
  • Deeper diegetic audio that reacts to proximity (RTPC‑like behavior).
  • Expand frame roster with branching/optional challenges.
  • Instrument analytics for time‑to‑advance, comfort toggles, and churn points.
